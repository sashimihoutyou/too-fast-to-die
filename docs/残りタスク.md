最終更新: 2026-06-28
対象: GDD分割後ドキュメント群、実装仕様書、現行コード/リソースの差分、プレイテスト所見

> `docs/archive/remaining_tasks.md` と `docs/archive/game_reassessment.md` の統合版を、2026-06-28時点の実装状況に合わせて再整理したもの。
> 優先度は「コアゲームループ」「説明と実挙動の一致」「プレイヤー判断に必要な情報」「コンテンツ量」の順で評価する。

## 凡例

- **[CRITICAL]** コアゲームループに直接影響する。これが欠けているとゲームとして成立しない
- **[HIGH]** プレイ体験と判断品質に大きく影響する
- **[MEDIUM]** 体験改善または開発効率改善
- **[LOW]** ポリッシュまたはコンテンツ拡充

## 1. 同行者システムのゲームループ統合 [CRITICAL]

### 今回着手済み

- `CompanionData.deck_card_ids` を追加。
- 全7タイプの同行者に専用カードIDを設定。
- `resources/cards/companions/` に同行者専用カード7枚を追加。
- 加入時に専用カードをデッキへ追加し、離脱/差し替え時に除去する処理を追加。
- 未実装だった技術者・密告者・犬・裏切り者の加入イベント `.tres` を追加。
- UI上の裏切り者予兆テキストを追加。
- 離脱報酬の説明と実処理を同期（商人=燃料8、戦闘型=アンコモンカード、難民=カルマ+15、技術者=上位バイクパーツ）。
- 同行者の加入/交代/離脱/離脱報酬をマップ帰還時に通知するキューを追加。

### 残作業

- 専用カード7枚の数値と実戦価値をプレイテストで調整する。
- 裏切り者の窃盗を「離脱時処理」だけでなく、専用イベント演出として見せるか決める。
- 技術者の「上位バイクパーツ」報酬が装備の上書きを伴うため、入手/装着確認UI（§6）と接続する。

## 2. 既存クエスト定義を実イベントへ接続 [HIGH]

### 現状

`resources/quests/` には以下4件が存在する。

- `devilf_pack.tres`
- `raider_pursuit.tres`
- `wanderer_clue_chain.tres`
- `neon_eden_escape.tres`

ただし、イベント側で `starts_quest` / `quest_outcome` を設定して実際に起動するのは `devilf_pack` のみ。

### 作業

- `raider_pursuit`、`wanderer_clue_chain`、`neon_eden_escape` の導入イベントを作る、または既存イベントへ `starts_quest` を追加する。
- クエスト開始後のアウトカムがボス修飾や遅延ペイロードに届くか確認する。
- 最低5件まで拡充する前に、既存4件をすべて「発火するクエスト」にする。

## 3. 走行パートイベント仕様の確定 [HIGH]

### 現状

GDD§17 は「ノード間移動時に30%の確率でミニイベント発生」としている。
現行実装は `MapGenerator` が EVENT ノードを生成し、そのノードに入った時だけイベント画面へ遷移する。

### 作業

- 仕様をどちらかに決める。
  - A: GDD通り、ノード間30%ミニイベントを追加する。
  - B: 現行通り、EVENTノードへ統合し、GDDの走行パート6種を travel event として扱う。
- Aを採用する場合、発生率30%をプレイテスト用にパラメータ化する。
- Bを採用する場合、GDD§17の文言をEVENTノード仕様に更新する。
- 未実装の走行パートイベント（エンジン異音、遠くの煙、汚染の風、行き倒れの旅人、野生の獣、好天）を `.tres` 化する。

## 4. イベント `.tres` 量産と選択肢の多様化 [HIGH]

### 現状

イベント総数は増えているが、クエストや同行者と強く結びつくイベントはまだ少ない。

### 作業

- 実装仕様書 S7-1（集落イベント4種）、S7-2（勢力イベント3種）、S7-3（キャラ固有イベント全PC分）を `.tres` 化する。
- 同行者の有無で選択肢が変わるイベントを優先する。
- `sets_flag` による後続イベント予約を活用し、1回の選択が2〜4ノード後に跳ね返る構成を増やす。
- イベント固有敵を `.tres` に設定する（`combat_enemy_ids` フィールドは実装済み）。
