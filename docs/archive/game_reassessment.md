# Road to Oasis — 企画再評価・実装比較・改善提案

調査日: 2026-06-25  
対象: `Too_Fast_to Die_gdd_v2.md`、実装仕様書、現行 Godot プロジェクト、データリソース

## 1. 総合評価

現状は「カードバトルローグライト + ロードムービー + サバイバル資源管理」という企画の核が実装に落ち始めており、キャラクター固有ゲージ、クエスト、同行者、追跡、信仰、汚染カード、バイクパーツ、セーブ/ロードなど、GDD上の主要システムの多くは基盤が存在する。

一方で、プレイヤー体験としては次の3点がボトルネックになっている。

1. **ランごとの報酬軸がカードに偏りすぎている**  
   アイテム/レリック相当の永続報酬が未実装で、ローグライトとしてのビルド幅が不足している。
2. **情報提示が不足している**  
   デッキ確認、カード詳細、強化差分、バイク耐久、イベント条件など、意思決定に必要な情報が画面に出きっていない。
3. **コンテンツ密度がシステム密度に追いついていない**  
   イベント・クエスト・フレーバーテキスト・固有敵指定など、既存システムを活かすデータ量と演出が不足している。

## 2. 企画の改善提案

### 2-1. 「燃料」の三重性をより強く押し出す

GDDの強みは、燃料が「移動」「戦闘」「通貨」を兼ねる点にある。現状実装では燃料表示・消費はあるが、プレイヤーに毎回ジレンマを迫るにはまだ弱い。

提案:

- ショップ価格、カード燃料コスト、移動消費、イベント選択肢を同じUI文脈で比較できるようにする。
- ノード間の距離に応じて移動コストを変え、遠いノードほど燃料を多く要求する。
- 同行者がいる場合は、護衛・食糧・積載の負担として移動コストを少し増やす。
- マップ上に燃料/水を発見できるノードを混ぜ、対象ノード上に獲得量を表示して「危険だが補給できる道」を選べるようにする。
- バイクを持たない覇者（コンカラー）は表示名とアイコンを「燃料」ではなく「水」に変える。ただし機能は燃料と同一にし、既存の資源ロジックを重複実装しない。
- 高燃料カードは単なる高火力ではなく、「次ノード到達リスクを受け入れる決断」として説明する。
- イベント報酬に「燃料は得るがヒート/カルマ/汚染が悪化する」選択を増やす。

### 2-2. レリック/アイテムを「旅の痕跡」として設計する

一般的なローグライトのレリックをそのまま導入すると世界観が薄まる。`ItemData` の基盤を活かし、アイテムを以下の2種に整理するのがよい。

- **消耗品**: 医薬品、燃料添加剤、即席爆薬、スティムなど。短期的な生存判断を作る。
- **遺物/装備品**: 旧世界のコンパス、宗教的護符、改造工具、変異革ジャケットなど。ランごとのビルド個性を作る。

### 2-3. イベントを「善悪」ではなく「誰に借りを作るか」で分岐させる

カルマだけでなく、派閥・同行者・追跡・信仰・オアシス情報が既にあるため、イベント設計は単純な善悪よりも「どの共同体と関係を持つか」に寄せると深くなる。

提案:

- 選択肢結果に `sets_flag`、クエスト進行、同行者加入、追跡増減を積極的に組み合わせる。
- 後続イベント予約を使い、1回の選択が2〜4ノード後に跳ね返る構成を増やす。
- イベント固有敵を指定できるデータ項目を追加し、戦闘も物語の続きにする。

## 3. 現状実装との比較

| 領域 | 企画上の狙い | 現状 | 修正方針 |
|---|---|---|---|
| カード戦闘 | キャラ別デッキと固有システムで差別化 | 固有ゲージ/獣/汚染などは実装済みだが、一部カード効果と強化値に未確認・不足が残る | カードデータ監査と固有カード効果のテストを優先 |
| 資源管理 | HP・燃料/水・医薬品・バイク耐久で旅の緊張感を作る | 距離別移動コスト、同行者による消費増、補給ノード表示、覇者の水表示は実装済み。医薬品使用の提示は弱い | 医薬品使用導線と燃料報酬イベントを追加 |
| ローグライト報酬 | カード、パーツ、アイテム、同行者でランの個性を作る | カード/パーツ/同行者は基盤あり。アイテムはデータクラスのみ | ItemDatabaseと入手・使用UIを追加 |
| イベント | 選択が後続イベント・フラグ・戦闘に影響 | フラグ・クエスト・予約は基盤あり。固有敵指定とコンテンツ量が不足 | EventChoiceData拡張とイベント増量 |
| メタ進行 | PC解放や周回目標を作る | 実装あり。GDDとの表記差分は要照合 | GDD/データ/表示文を同期 |
| 情報提示 | 意思決定に必要な比較情報を見せる | デッキ確認、詳細ポップアップ、強化差分などが不足または一部実装 | 共通カードプレビューを作る |

## 4. 修正案ロードマップ

### 最優先: プレイ判断を阻害する欠陥

1. カード強化値の監査と補完
2. 休憩画面・報酬画面・ショップでの強化/購入前プレビュー統一
3. マップ画面からのデッキ閲覧
4. バイク耐久と医薬品使用導線の明確化

### 次点: ランごとの変化を増やす

1. アイテム/レリックの最小実装
2. ショップ、戦闘報酬、イベント報酬へのアイテム導入
3. バイクパーツの入手・装着導線の拡充
4. イベント固有敵指定

### コンテンツ拡充

1. travel/faction イベントを各Actに最低2件ずつ追加
2. クエストを最低5件まで拡充
3. キャラクター固有イベントを各PC 2件以上に揃える
4. 主要カードから順にフレーバーテキストを追加

## 5. ファイル整理方針

- 進捗管理は `docs/remaining_tasks.md` に一本化する。
- ルート直下のキャラクター画像は `assets/characters/` に移動し、Godot import の `source_file` も移動後パスに更新する。
- 完了済みの古い作業リストは削除し、必要な内容のみ残作業一覧へ反映する。
