# Too Fast to Die: Road to Oasis — ロア作成ルール

この文書は、敵・集落・NPC・勢力拠点・イベント背景・アイテム由来などのロアを新規作成/改修する際の必読ルールである。

ロア作成時は、まず本書を読み、必要に応じて `docs/design/gdd-core.md`、`docs/design/gdd-events.md`、`docs/design/gdd-narrative.md`、`docs/design/gdd-characters.md`、`docs/design/gdd-companions.md`、`docs/lore/world-rules.md` を参照する。

---

## 1. 基本方針

本作のロアは、「一目でイカれた世界だとわかる」ことを目指す。ただし、ただ奇抜なだけにしない。

良いロアは、外から見ると狂っているが、その世界の中では資源・恐怖・信仰・制度・生存戦略として成立している。

例：

- ガソリンを聖なる血と呼ぶV8カルトは、燃料が通貨であり命綱である世界では成立する
- ラケダイモーンの奴隷医療は、倫理的には最悪だが、人口と労働力を資源視する国家では合理性を持つ
- ニューエデンの借金娼館は、自由社会の顔を保ったまま搾取する制度として成立する

NG：

- 理由なくモヒカンが叫んでいるだけ
- 旧世界パロディをそのまま置いただけ
- 「狂っている」と説明するだけで、生活・制度・戦闘・選択肢に反映されない

---

## 2. ロアの核に置く四資源

新しい敵・集落・NPC・勢力拠点を作る時は、必ず以下のうち最低1つを核にする。

| 資源 | ロア上の意味 | 使い方 |
|---|---|---|
| 水 | 生存、清潔、医療、農業、権力 | 水源を守る掟、浄水装置の支配、洗えることの贅沢 |
| 燃料 | 移動、戦闘、通貨、信仰 | ガソリン取引、燃料税、V8儀式、移動不能の恐怖 |
| 人口 | 労働力、繁殖、兵士、家族、商品 | 奴隷制、婚姻仲介、孤児保護、子供の扱い |
| 情報 | オアシス、技術、地図、秘密、脅迫材料 | ノウアスフィア報告書、偽情報、情報屋、読める者の特権 |

「何を奪い合っているのか」が見えないロアは弱い。

---

## 3. 勢力は区画ではなく関係で描く

現在のマップ方針では、Act全体を単一勢力圏として固定しない。同じステージ内に、ニューエデン側集落、ラケダイモーン前哨基地、V8カルトのピットイン、ダストランナー停泊地、チェインリンク隊商、ノウアスフィア隠し施設などが混在する。

ロア作成時は、その場所が「どの勢力の土地か」だけでなく、「他勢力とどう接しているか」を決める。

最低限決めること：

- 誰が表の支配者か
- 誰が裏で物資・人・情報を流しているか
- プレイヤーがどちらに寄ると誰に嫌われるか
- 高カルマ/低カルマで態度が変わるか
- PC固有の因縁があるか

例：

- ニューエデン側集落だが、裏口ではチェインリンクが借金者を買っている
- ラケダイモーン前哨基地だが、医者はノウアスフィア公開派の流出技術を使っている
- サンドバイパーズ検問所だが、V8カルトだけは「巡礼料」で通す

---

## 4. 「狂気」は生活様式に落とす

ロアの異常性は、見た目・口調・制度・取引・戦闘・選択肢に分散させる。

作成時のチェック：

- 見た目だけで所属や異常性がわかるか
- その異常性は毎日の生活でどう維持されているか
- 旅人に何を要求するか
- 断ると何が起きるか
- 受け入れると何を失うか

良い例：

- 水を神聖視する集落では、来客に水を飲ませず、額に一滴だけ垂らす。飲む水は共同体の許可が必要
- ラケダイモーン農場では奴隷に番号が刻まれ、医療記録だけは異様に清潔な字で残っている
- V8カルトのピットインでは燃料缶が祭壇に並び、空の缶も遺骨のように扱われる

## 5. 性的ロアの作成ルール

本作の性的要素は、ご褒美や挿入用の飾りではない。人口が減り、水と燃料が命綱になった世界で、身体・生殖・快楽・搾取・保護・取引が社会制度や生存戦略に組み込まれていることを示すために存在する。

性的ロアを作成する時は、必ず `docs/lore/world-rules.md`、`docs/writing/context-types.md`、必要なら `docs/writing/pc-profiles.md` を参照する。実際のシーン本文を書く場合は `.agents/skills/tftd-writing/SKILL.md` の濃淡コントロールと合意/非合意ルールに従う。

### 5-1. 性的ロアの核

性的ロアは、以下のうち最低1つに接続する。

| 核 | 内容 | 例 |
|---|---|---|
| 人口 | 繁殖、子供、婚姻、家族、労働力 | ラケダイモーンの繁殖政策、ダストランナーの婚姻仲介 |
| 取引 | 燃料、水、ネオン、医療、通行料との交換 | サンドバイパーズの通行料代替、ニューエデンの借金娼館 |
| 支配 | 奴隷制、見世物、懲罰、服従儀式 | チェインリンクの奴隷市場、レイダー拠点の戦利品扱い |
| 依存 | ネオン、快楽、離脱症状、身体の回路化 | ホタル、ネオン中毒者、歓楽街の元従業員 |
| 信仰 | 儀式、共同体、禁欲/放縦、身体の聖別 | V8カルトの加速儀式、燃料信仰との混線 |
| 生態 | 変異獣、ミュータント、繁殖適応、捕食 | ブルードマザー、ノウアスフィア研究記録 |

「裸にすれば世紀末らしい」「過激にすれば荒廃感が出る」という発想は禁止。何の資源・制度・恐怖に繋がるのかを先に決める。

### 5-2. 合意/非合意は曖昧にせず分類する

ロア設計時点で、その性的状況がどの分類かを明確にする。

| 分類 | ロア上の扱い |
|---|---|
| 自発的享楽 | 一時の解放。代償や依存性を持たせる。カルマ影響なしが基本 |
| 取引的合意 | 合意はあるが、飢え・借金・ネオン・燃料不足で選択肢が狭い。境界の曖昧さを書く |
| 制度的強制 | 個人ではなく制度が身体を資源化する。ラケダイモーン、奴隷商、繁殖施設向き |
| 暴力隣接 | レイダー、奴隷制、報復、捕獲。カルマ・脱出・逆転・後続影響を必ず持たせる |
| 目撃 | PCが何を見るか、介入できるか、見なかったことにするかを主軸にする |
| 変異獣/動物 | 善悪でなく生態・本能・共生・恐怖として扱う。単なる刺激物にしない |

非合意や搾取を扱う場合、被害者を舞台装置にしない。恐怖、怒り、諦め、無感覚、生存計算など、個体としての反応を持たせる。PCが加害側に回る場合はカルマ・同行者反応・後続イベントに反映する。

### 5-3. 勢力別の性的ロア

勢力ごとの性文化は `docs/lore/world-rules.md` を正とする。ロア作成時の要点は以下。

| 勢力/場所 | ロア上の要点 |
|---|---|
| ニューエデン | 自由と搾取の同居。歓楽街、借金契約、顧客名簿、ネオン、闘技場の見世物性 |
| ラケダイモーン | 繁殖と医療の管理。番号、記録、検査、奴隷妻、子供の国家育成 |
| V8カルト | 共同体、孤児保護、信仰と身体の混線。禁欲と儀式的放縦の両方があり得る |
| サンドバイパーズ | 通行料、力、取引。身体も燃料も同じ「通行の対価」になり得る |
| チェインリンク | 奴隷売買、借金、移送、商品化。戦闘より帳簿と鎖の組織 |
| ダストランナー | 婚姻仲介。善意の縁談と半売買の境界が曖昧 |
| ノウアスフィア | 直接行為より記録・観察・繁殖適応研究。感情を排した文体が恐怖になる |
| 独立集落 | 集落ごとの掟。水源、血縁、人口維持、外部婚姻、隠された被害 |

### 5-4. 変異獣・動物・ミュータントの扱い

人間以外との性的ロアは、必ず生態・研究・信頼・恐怖のどれかに接続する。

| 類型 | 使い方 |
|---|---|
| 変異獣の繁殖 | ホラー/生態。ブルードマザーのように失踪事件、巣、救助、脱出へ接続 |
| 調教師と相棒 | 信頼・共生・訓練の延長。擬人化しすぎず、単純な獣姦として処理しない |
| ノウアスフィア研究 | 冷徹な記録。種間繁殖適応や変異生態を情報ノードに接続 |
| ミュータント化した人間 | 差別・身体変化・欲望の変質。人間性を失ったかどうかを安易に断定しない |

動物や変異獣のロアは「人間社会の規範が壊れたから何でもあり」ではなく、「この個体/種がどう生き、どう繁殖し、人間社会をどう歪ませたか」を決める。

### 5-5. 年齢と弱者の扱い

子供や未成年を性的対象として扱わない。子供に関するイベントは、人口減少、孤児保護、奴隷取引、家族の喪失、国家育成などのロアとして扱い、直接的な性的描写を置かない。

成人でも、保護対象・奴隷・救出直後の人物・依存症者などは自由な選択ができない場合がある。合意と搾取の境界を曖昧にしたまま「合意だから問題なし」と処理しない。曖昧さそのものをロアと選択肢にする。

### 5-6. 実装へ落とす時の必須要素

性的ロアをイベントや敵に落とす場合、以下を決める。

- PCが介入できるか、目撃だけか、後で知るだけか
- 介入する/しない/利用する/取引する選択肢があるか
- カルマ、信仰度、ヒート、清明度、同行者反応に影響するか
- 被害者/相手が後続イベントで再登場するか
- その場の描写濃度は `docs/writing/context-types.md` のどれに該当するか
- プレイヤー向けUIに未実装の性的制度や効果を書いていないか

## 6. 敵ロアの作成ルール

敵は「戦闘する障害」であると同時に、その土地の生活の結果である。

敵を作る時は以下を決める。

| 項目 | 内容 |
|---|---|
| 生存理由 | なぜこの世界で生き残れているか |
| 資源との関係 | 水・燃料・人口・情報のどれを奪う/守る/利用するか |
| 見た目の一撃 | 一目で何者か分かる視覚要素 |
| 戦闘上の個性 | 行動パターン、状態異常、報酬にどう出るか |
| 非戦闘接点 | 交渉、イベント、集落被害、PC固有反応 |

変異獣は「放射線で強くなった」だけにしない。繁殖できるのか、群れるのか、人間が利用できるのか、人間社会をどう変えたのかを決める。

人間敵は「悪党」だけにしない。所属、食い扶持、恐怖、掟、上納先を決める。

機械敵は「旧世界の残骸」だけにしない。何を守る命令が残っているのか、誰が再起動したのか、壊れた認識で何を敵とみなすのかを決める。

## 7. 集落ロアの作成ルール

集落は人口20〜50人程度を基本とし、必ず「一つの深刻な問題」を持つ。

最低限決めること：

- 水源または食料源
- 外部との取引手段
- 守っている掟
- 隠している恥/罪/病
- プレイヤーが介入できる選択肢
- 介入しなかった場合の後続影響

集落の問題は、単なる依頼ではなく、その集落の価値観を見せるものにする。

例：

- 浄水装置を直せるのは一人だけだが、その人物は借金でニューエデンに売られる寸前
- 疫病集落がラケダイモーンの保護を求めており、受け入れると全員助かるが自治を失う
- 子供を外に出さない集落。理由は過保護ではなく、チェインリンクに子供の人数を知られないため

## 8. NPCロアの作成ルール

NPCは「設定説明役」にしない。ロアは、その人物の行動・持ち物・口調・選択肢で見せる。

最低限決めること：

- 何を欲しがっているか
- 何を恐れているか
- 何を売れる/差し出せるか
- どの勢力に借りがあるか
- PCごとに態度が変わるか

NPCの善悪は固定しない。助ける価値がある人物でも、過去に誰かを売っていることがある。敵対する人物でも、集落の子供だけは守っていることがある。

## 9. パロディと参照元の扱い

`MADMAX`、`北斗の拳`、`Fallout` 的な空気は参照してよい。ただし固有要素をそのまま移植しない。

使ってよいもの：

- 荒野の信仰
- 物資をめぐる誇張された制度
- 奇抜な武装や髪型
- 死と隣り合わせの笑い
- 旧世界文化の誤読

避けるもの：

- 既存作品の名前・組織・台詞の直写し
- 「ヒャッハー」だけで成立する敵
- Fallout風のVaultをそのまま増やす
- 北斗の拳の拳法をそのまま量産する

本作固有の置き換え軸は、燃料、水、人口、情報、ネオン、V8信仰、オアシスである。

## 10. タイムライン整合

現在は崩壊から32年後。

ロアを追加する時は、年齢・世代・出来事の前後関係を必ず確認する。

- 戦前世代：核戦争時に子供だった現在の中年層
- 戦後世代：荒廃した世界しか知らない若者
- 32年以上前の記憶を直接持つ若者は存在しない
- 戦前施設を「知っている」ことと「伝聞で信じている」ことを区別する
- 誰が誰を知っている/知らないかは、移動経路と隠遁理由で説明する

矛盾が出た場合、設定を消す前に、隠された家族、奴隷商による移送、放浪者同士のすれ違い、ラジオ通信の誤情報など、世界観内で成立する整合解を探す。

## 11. 実装・データとの同期

ロアを実装データに落とす場合、説明文だけで終わらせない。

- 敵ロアを追加したら、敵データ、遭遇フレーバー、報酬、戦闘行動のどれに反映するか決める
- 集落ロアを追加したら、イベント選択肢、後続フラグ、カルマ変動、関係値変動を決める
- 勢力拠点を追加したら、マップノードの勢力/施設種別、ショップ価格、イベント抽選に反映する
- GDDと `.tres` に同じ数値や固有名がある場合は同期する
- 未実装の効果をUI説明やカード説明に書かない

---

## 12. ロア作成チェックリスト

新規ロアを出す前に、以下を確認する。

- 水・燃料・人口・情報のどれに関わるか
- 性的ロアを含む場合、人口・取引・支配・依存・信仰・生態のどれに接続するか
- 合意/取引的合意/制度的強制/暴力隣接/目撃/変異獣のどれに分類されるか
- どの勢力と利害関係があるか
- 一目でわかる視覚要素があるか
- 「狂っているが機能している」理由があるか
- プレイヤーが選択で関われるか
- 受け入れる/拒否する/無視する/利用する分岐を作れるか
- PC固有反応を作る価値があるか
- 32年後のタイムラインと矛盾しないか
- 既存のGDD、イベント、`.tres` と名前・数値が衝突しないか
- 実装予定のない効果や制度を、プレイヤー向け説明に書いていないか

このチェックを通らないロアは、見た目が面白くても一度保留する。
