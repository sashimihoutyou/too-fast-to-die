# サブストーリー設計：ビッグリック掃討

Act2に配置する複数アジト連鎖破壊型のサブストーリー。`docs/scenarios/substories/devilf-pack.md` の
**「名前付きアウトカム＋遅延ペイロード」モデル**を踏襲しつつ、単発クエストでは表現できない
「複数拠点の連続破壊→頭領捕縛→処断分岐」という多段構造を追加する。

> **実装状況**: 本書は設計のみ。§9 に必要な実装（既存機構の再利用と新規拡張の切り分け）を記す。
> `.tres` 相当のコードブロックは**提案スキーマ**であり、実ファイルは未作成。

---

## 0. 設計原則（devilf-pack から継承・追加分）

- アウトカムは名前付き（`hunt` / `ignore` / `lapse`）。空アウトカム禁止（no-op禁止）。
- 道徳コストはテレグラフし、カルマ本体は遅延加算。**カルマ具体値は仮置き**（本実装までに調整）。
- 処理後は本流（通常のマップ進行）へ再収束。
- アウトカムの出し分けはPCで非対称にする（ホーネット/ホタルの固有反応）。
- **新規**: 単発の「討伐/放置」二択ではなく、複数アジト（拠点）を破壊するごとに**都度の資源選択**（燃料を割いて解放女性を送るか）が発生する、繰り返し構造を持つ。

## 1. 勢力ロア：ビッグリック

| 項目 | 内容 |
|---|---|
| 構成 | 全員男性。統率者は「ヒュージ・ディックソン」 |
| 生存理由 | 棄民・貧農崩れの寄せ集め。まともな集落からは相手にされず、行き場を失った者たちが数だけは膨れ上がった。組織としての規律はコカトリスほど無い |
| 資源との関係 | 人口（食料と若い女を強奪の主目的とする。特に若年女性）。集落を襲うのは物資目当てというより、行き場のない絶望と憎悪の発散が主目的で、資源効率で動いていない点がコカトリス等の「合理的レイダー」と一線を画す |
| 動機の多元性 | 一枚岩ではない。「世界に見捨てられ、世界を憎んでいる」「単にタガが外れた暴徒で後先を考えない」「性暴力そのものへの中毒者で自制が利かない」が混在する。統一された思想やイデオロギーは無い |
| 評判 | 他レイダー団からも恐れられ蔑まれる。彼らが死んで喜ぶ者は多いが、悲しむ者は一人もいない |
| コカトリスとの関係 | 対立・コカトリス優勢。過去にビッグリックが（相手と知らず）コカトリス構成員を襲い、報復でアジトを一つ潰され構成員多数を失った。以来ビッグリックはコカトリスとの遭遇を避けている。直接対決はしていない均衡状態 |
| 見た目の一撃 | 統率された制服的意匠を持たない寄せ集めの装備。ディックソンのみ突出した体格と装飾で頭領と分かる |
| 戦闘上の個性 | 雑兵は数を頼みに一斉に襲うタイプ（数的優位・個の練度は低い）。ディックソンは高HP・拘束系の行動パターン |

`docs/lore/lore-rules.md` §1 の「狂っているが機能している」を満たす鍵は、**彼らが世界の残酷さの結果であって原因ではない**という描き方にある。構成員数の多さそのものが「この世界にはそれだけ食い詰めた絶望的な人間がいる」という世界観の証言になる。

## 2. クエスト定義（QuestData 拡張案）

既存 `QuestData` は単一目的（`objective_match` の敵タグ撃破カウント→単一ボス）を前提にしている。本クエストは**アジト破壊(3件・非ボス)→頭領捕縛(ボス)**という2段構造だが、これは既存スキーマにそのまま乗る（アジトの雑兵に `big_rick` を含むIDを振り、カウント3でhunt完遂→その後にボス `huge_dickson` が出現する専用ノードが解禁される、という順序）。

| 項目 | 値 |
|---|---|
| id | `&"big_rick"` |
| title | ビッグリック掃討 |
| required_act | 2 |
| node_limit | 14（3アジト分の探索猶予） |
| intro_event | `&"big_rick_intro"` |
| objective_match | `&"big_rick"` |
| objective_count | 3（アジト3件） |
| boss_target | `&"huge_dickson"` |
| outcomes | `hunt` / `ignore` / `lapse` |

```
# 提案スキーマ（QuestData）
id             = &"big_rick"
title          = "ビッグリック掃討"
summary        = "悪名高いレイダー団の拠点を破壊し、頭領ディックソンの処遇を決める"
required_act   = 2
node_limit     = 14
intro_event    = &"big_rick_intro"
objective_match = &"big_rick"
objective_count = 3
boss_target    = &"huge_dickson"
payload_event  = &"big_rick_judgment"
payload_outcomes = [&"hunt"]
payload_min_gap  = 0
```

### 導入経路（PC共通、フレーバーのみ分岐）

イベント本文はPCの因縁経路によって差し替える（実装は同一 `big_rick_intro` の本文をPC別に分岐、または導入イベントを3種用意し同一 `starts_quest` へ収束）。

- 直接被害: PCまたは同行者がビッグリックに襲撃される
- 目撃: 破壊された集落の跡地でビッグリックの犯行と知る
- 依頼: 被害者からの依頼を受ける（生存者NPCが仲介）

いずれも選択肢は「掃討を決意する」（`quest_outcome=hunt`）／「関わらない」（`quest_outcome=ignore`）の二択。`ignore` はカルマ変動なし・以後この勢力に関与しない（devilf の `ignore` と同型、authoredな空でない帰結として「彼らはそのまま荒野で被害を広げ続ける」ことを暗黙の代償にする）。

## 3. アジト破壊：都度の資源選択（新規パターン）

アジト（`big_rick` タグの非ボス戦闘ノード、Act2内に3箇所）を撃破するたびに、**即時の小イベント**（既存の戦闘後報酬画面 or 軽量EVENT差し込み。§9で機構を検討）が発生する。

| 選択 | 効果 | 備考 |
|---|---|---|
| 追加燃料を割いて解放女性を送る | `fuel_change = -2`、`karma_change = +1`（即時・小） | 送られた女性は`sets_flag`で個体を記録しない（軽量化のため）。後続の**モブ同行者/NPCの経歴にランダムでビッグリックの生存者だった旨が出現する**抽選プールに1件加算する |
| 燃料を割かず自力で行かせる | 効果なし（燃料温存） | 「送り届けられなかった」という軽い忌避のみ。抽選プールには加算しない＝彼女たちのその後は語られない（暗黙の重さ） |

3回とも同一の二択（毎回異なる文面のバリエーションを数種用意し単調さを避ける）。**no-op禁止**を満たすため、両者とも数値差分を必ず持たせる。カルマの具体値は仮置き。

## 4. 頭領捕縛と処断分岐

3アジト破壊完了後、Act2内に専用ノード（`huge_dickson` をボスとする戦闘）が解禁される。撃破＝殺害ではなく**捕縛**として扱う（戦闘勝利後の説明文で「殺さず取り押さえた」と明示）。

勝利直後、`big_rick_judgment` を強制発火（`payload_min_gap=0`）。処断先を選ぶ。

| 選択 | ニューエデン | ラケダイモーン | ノウアスフィア | 噂/後続 | カルマ(仮) |
|---|---|---|---|---|---|
| ニューエデンで縛り首 | **+1** | ±0 | ±0（無関心） | なし | 上昇（仮） |
| 荒野に追放する | ±0 | ±0 | ±0 | なし | 微増（仮） |
| ラケダイモーンに連れていき石打ち | ±0 | **+1** | ±0 | 報酬あり | 微増（仮） |
| アジト跡地で逆さ吊り腹裂き惨殺 | ±0 | ±0 | ±0 | **`sets_flag = &"dickson_massacred"`**。アンビエントフラグメント/一部NPC台詞で後日話題に出る | 上昇（仮、皮肉枠） |

ノウアスフィアは一貫して無風（戦前テクノロジー以外に関心を示さないという性格描写であり、実装漏れではない）。縛り首/石打ちは「その勢力の面前で処断する＝直接的功績」という理由づけのため、行った側の勢力のみ+1し反対側は変動しない。追放と惨殺はアジト跡地（第三者不在）で完結するため、どちらの勢力好感度も動かない。

`docs/scenarios/substories/devilf-pack.md` §6 と同じ理由で、**カルマは処断選択時に即時本体を加算してよい**（遅延ペイロードでの分岐そのものが最終アウトカムであり、これ以上隠す情報が無いため）。

## 5. PC別の差分

| PC | 差分 |
|---|---|
| 元レイダー ホーネット | ヒート値が一定以上の場合、処断選択に追加の残虐行為オプションが解禁される。選択後の一文で「これをやるのは初めてじゃない」と仄めかす——コカトリス時代、レイヴンの下で下した命令の記憶に触れる伏線（本文では明言しない）。既存の§7-2「ヒート帯に応じてイベント選択肢が制限される」システムをそのまま再利用する |
| 享楽者 ホタル | 撃破直後、処断より先に一言選択肢が入る：「大きい！　殺しちゃう前に記念にとっておこう」。トロフィー的な軽さで彼女の狂気を表現する。性的シーンにおける「NEEDINGとENJOYINGの混同禁止」ルールとは別カテゴリ（暴力・猟奇のトーン）のため抵触しない |
| カルティスト アータル | ディックソンの処遇を「V8への生贄」的な語彙で解釈する台詞差分（機械的効果は他PCと同一） |
| 放浪者 ウェズリー | 崩壊直後の飢餓・喪失の記憶と重ね、追放ルートに強い忌避を示す一文差分 |
| 調教師 ミーシャ | 元被奴隷児としての視点から、女性たちの解放を最優先する台詞差分。惨殺ルートには「けだものはけだものだが、これは違う」という距離感 |
| 覇者 | 「弱きを助け強きをくじく」既定の性格どおり、縛り首か惨殺のどちらかに自然と誘導される（知性面の読み不足で交渉/取引寄りの選択肢が見えにくい） |

## 6. アンビエントフラグメント連携（惨殺ルートの噂）

`sets_flag = &"dickson_massacred"` を `scripts/systems/ambient_fragment.gd` の選出条件に追加し、以後のノードで低確率抽選されるフラグメントプールに「ディックソンの末路を語るNPCの噂」を1〜2件加える。`once_per_run` 相当の重複防止は既存機構をそのまま使う。惨殺以外の処断では噂プールは変化しない（静かに終わる処断は世間の記憶に残らない、という非対称性そのものが演出）。

## 7. 本流への再収束

```
[big_rick_intro]
   ├ hunt ─→（アジト×3破壊・都度の燃料選択）─→ [huge_dickson 捕縛戦] ─→ [big_rick_judgment 処断4択] ─→ 通常進行
   ├ ignore ──────────────────────────────────────────────→ 通常進行（以後関与なし）
   └ lapse（期限切れ）────────────────────────────────────→ 通常進行（アジトは残ったまま風化）
```

## 8. 実装（新規に必要な機構）

devilf-packで確立済みの機構（QuestData/QuestManager本体、EVENTノードのペイロード強制発火、`starts_quest`/`quest_outcome`/`sets_flag`）は**そのまま再利用可能**。以下は本クエスト固有で新規に要る部分。

1. ⬜ **ボス撃破によるペイロード装填**：現状 `QuestManager.record_outcome()` は選択時にのみペイロードを装填する。本クエストは捕縛（ボス撃破）契機で装填する必要があるため、`on_enemy_defeated()` に `data.is_boss and data.id == def.boss_target` の分岐を追加し、`_armed` に即時装填するフックが要る。
2. ⬜ **hunt完遂後にのみ出現する専用ボスノード**：`huge_dickson` は通常のAct2固定ボスではなく、hunt完遂（`progress >= objective_count`）を条件にマップ上へ解禁される専用ノード。マップ生成側（`map_screen.gd` / マップ生成ロジック）に「クエスト完遂条件を満たすまで非表示、満たしたら次の適切な位置に挿入」という条件付きノード生成が必要（デビルフの固定Actボスとは異なる新パターン）。
3. ⬜ **アジト撃破ごとの軽量選択イベント**：3回とも同一クエスト内で発生する繰り返し二択。既存の戦闘後報酬画面に差し込むか、`payload_outcomes` を拡張して複数回武装できるようにするかは要検討（現状 `_armed` はList管理なので複数エントリの同時武装自体は可能。ただし同一 `event_id` の複数回発火は `already_seen` フラグで1回しか出せないため、3種の別イベントID、または `payload_event` を都度動的に切り替える拡張が要る）。
4. ⬜ **勢力好感度サブシステム**（ニューエデン/ラケダイモーン/ノウアスフィアの-1/0/1/2値）：新規Autoload（例: `FactionReputation`）または `GameManager` への辞書追加。本クエストはこの最小実装（3勢力・4値のint、変動APIのみ）があれば足りる。UI表示・価格連動等の消費側は別途。
5. ✅ アンビエントフラグメントの条件フック追加は既存 `event_flags` ベースの選出ロジックをそのまま利用可能。

## 9. ロア作成チェックリスト（`docs/lore/lore-rules.md` §12）確認

- 水・燃料・人口・情報のどれに関わるか → 人口（略奪・解放）、燃料（護送コスト）
- 性的ロアの核 → 支配（奴隷的拘束）、依存（中毒動機の一部）
- 合意/非合意分類 → 暴力隣接（カルマ・脱出・逆転・後続影響あり）
- 勢力関係 → コカトリスとの対立・力関係、ニューエデン/ラケダイモーン/ノウアスフィアへの処断分岐
- 一目でわかる視覚要素 → 統率なき寄せ集め装備＋頭領ディックソンの突出した体格
- 「狂っているが機能している」理由 → 棄民・貧農の成れの果てという世界の残酷さの証言として機能
- プレイヤーが選択で関われるか → 掃討可否、都度の燃料選択、処断4択
- 受け入れる/拒否する/無視する/利用する分岐 → hunt/ignore/lapse＋処断4択
- PC固有反応 → ホーネット(ヒート連動+過去仄めかし)、ホタル(トロフィー台詞)、他4PC台詞差分
- タイムライン矛盾なし（32年後の世界観に矛盾する固有設定なし）
- 実装予定のない効果をUIに書かない → 好感度システムは§8で最小実装を明記、未実装のまま数値だけ提示しない
