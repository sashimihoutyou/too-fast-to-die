# Road to Oasis — 統合残作業リスト

最終更新: 2026-06-25  
対象: GDD v2、実装仕様書、現行コード/リソースの差分

> 旧 `作業予定.md` と旧 `docs/remaining_tasks.md` を統合した現行版。完了済み項目は削除し、未完了・要検証項目だけを残す。

## 凡例

- **[CRITICAL]** コアゲームループに直接影響する
- **[HIGH]** プレイ体験と判断品質に大きく影響する
- **[MEDIUM]** 体験改善または開発効率改善
- **[LOW]** ポリッシュまたはコンテンツ拡充

## 1. コア不具合・データ整合

### 1-1. [CRITICAL] カード強化値の監査と補完

現状: `CardData` は `upgraded_damage` / `upgraded_block` を参照し、休憩画面も強化差分を表示できる。ただし、多数の `.tres` で強化後数値が 0 のまま残っている可能性がある。

作業:

- `resources/cards/**/*.tres` を走査し、強化対象なのに `upgraded_damage` / `upgraded_block` が未設定のカードを洗い出す。
- GDD/実装仕様書の強化値と照合し、`.tres` に反映する。
- 強化値が数値ではなく効果変化の場合は `upgrade_description` に明記する。

### 1-2. [HIGH] GDD・実装仕様書・データの数値同期

現状: AP/ターン、PC初期値、メタアンロック条件など、文書とコード/データの値が一致していない可能性がある。

作業:

- 正式値を現行コード優先で決め、GDDと実装仕様書を更新する。
- `resources/characters/*.tres` と `autoload/meta_progression.gd` の表示文を同期する。

## 2. アイテム/レリックシステム

### 2-1. [CRITICAL] ItemDatabase とインベントリの実装

現状: `ItemData` は存在するが、`.tres`、Database Autoload、インベントリ、使用導線がない。

作業:

- `resources/items/consumable/` と `resources/items/relic/` を作成する。
- `autoload/item_database.gd` を作成し、`project.godot` に登録する。
- 所持数・スタック・使用タイミング・セーブ/ロードを実装する。

### 2-2. [HIGH] アイテム入手/使用UI

作業:

- ショップ、戦闘報酬、イベント報酬にアイテムを追加する。
- 戦闘中の消耗品使用UIを追加する。
- マップ画面で医薬品/消耗品を使う導線を追加する。
- レリック一覧をHUDまたはポップアップで確認可能にする。

## 3. 情報提示・カード閲覧

### 3-1. [HIGH] 共通カード詳細ポップアップ

作業:

- 戦闘、休憩、ショップ、報酬、デッキ閲覧で使えるカード詳細ポップアップを作る。
- `CardData` に `flavor_text` を追加し、詳細表示に含める。
- デバッグエディターの長文プロパティに `flavor_text` を追加する。

### 3-2. [HIGH] マップ画面のデッキブラウズ

作業:

- マップHUDに「デッキ確認」ボタンを追加する。
- 所持カード一覧をポップアップ表示する。
- カード詳細ポップアップを再利用する。

### 3-3. [MEDIUM] 強化・報酬・購入時の比較表示

作業:

- 休憩画面の強化候補で、強化前/後の数値差分と効果差分を常に表示する。
- 戦闘報酬とショップにも詳細プレビューを適用する。

## 4. イベント/クエスト拡張

### 4-1. [MEDIUM] イベント固有敵指定

現状: イベント戦闘はActからランダム敵を選ぶだけで、イベント固有の敵を指定できない。

作業:

- `EventChoiceData` に `combat_enemy_ids: Array[StringName]` を追加する。
- `event_screen.gd` で指定がある場合は該当敵を優先する。
- 救助・討伐・派閥イベントに固有敵を設定する。

### 4-2. [LOW] イベントとクエストのコンテンツ拡充

作業:

- travel/faction イベントを増やす。
- クエストを最低5件まで増やす。
- キャラクター固有イベントを各PCで偏りなく追加する。

## 5. 資源管理・バイク・同行者

### 5-1. [MEDIUM] 燃料/水システムの追加調整

現状: 距離別移動コスト、同行者による消費増、補給ノード、覇者の水表示は実装済み。残りはデータ量とイベント連携。

作業:

- 燃料/水補給ノードの出現率と獲得量をプレイテストで調整する。
- 燃料/水報酬イベントを増やし、危険な補給ルートを選ぶ意味を強める。
- カード燃料コスト、ショップ価格、移動コストのバランスを同じ基準で再計算する。

### 5-2. [MEDIUM] バイク耐久の導線強化

現状: マップHUD表示と修理処理は実装済み。残りは移動・故障ルールの確定。

作業:

- 移動時の耐久消費と耐久0時ペナルティを最終設計として固定する。
- 修理・パーツ装着・耐久上限の説明をUIで見せる。

### 5-3. [MEDIUM] バイクパーツ入手/装着UIの拡充

作業:

- ショップ以外の入手経路を追加する。
- 装備中パーツとステータス補正を確認できる画面を作る。

### 5-4. [LOW] 同行者効果の追加検証

作業:

- 犬/密告者の看破効果がクエスト/イベント条件に十分反映されているか確認する。
- 同行者離脱時の報酬/ペナルティをイベント演出と同期する。

## 6. キャラクター固有カード効果の検証

### 6-1. [MEDIUM] ビーストマスターカード効果

作業:

- `bm02`〜`bm09` の全効果が実装・データ設定・UI表示で一致しているか確認する。
- `bm08` の獣側ブロックなど未設計部分を決める。

### 6-2. [MEDIUM] コンカラーのオーラ蓄積データ

作業:

- コンカラー専用カードの `.tres` に `status_effect = &"aura"` と `status_stacks` が適切に設定されているか確認する。
- オーラ消費カードの説明文と実挙動を同期する。

## 7. ポリッシュ

### 7-1. [LOW] オーディオシステム

作業:

- `autoload/audio_manager.gd` を追加する。
- BGM/SEの基本再生、シーン別BGM、カード/ダメージ/勝利SEを実装する。

### 7-2. [LOW] フレーバーテキスト執筆

作業:

- 主要カードから順に `flavor_text` を追加する。
- キャラ固有カードはPCの口調・背景と一致させる。

## 8. 開発者向け改善

### 8-1. [MEDIUM] デバッグリソースブラウザ改善

作業:

- フォルダ選択時にステータスバーへパスを表示する。
- 新規作成時のデフォルト保存先に選択中フォルダを使う。

### 8-2. [LOW] `scripts/systems/` の扱いを決める

現状: `.gitkeep` のみ。`combat_manager.gd` が肥大化しているため将来的な分離候補。

作業:

- 直近で使わないなら削除する。
- 分離する場合は、キャラ固有システム単位で移す。
