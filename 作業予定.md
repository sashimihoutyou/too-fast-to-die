## 2. イベントシステム（検証・改善） [HIGH]

### 現状
- 基盤は機能している: EventData/EventChoiceData/EventManager/QuestManager 全て動作
- 23イベント .tres ファイル（settlement 8、faction 3、character 8、travel 4）
- 後続イベント予約（`payload_only` + QuestManager の `_armed` 配列）は **動作する設計**
- ただし現在 `record_outcome()` でペイロード装填するのは `outcome == &"poison"` **のみハードコード**

### 改善・検証項目
- [ ] **2-1. 後続イベント予約の汎用化**
  - `record_outcome()` L37: `if outcome == &"poison"` のハードコードを、QuestData のフィールドで制御する汎用設計に変更
  - 例: `QuestData` に `payload_outcomes: Array[StringName]` を追加し、マッチするoutcome全てでペイロード装填
- [ ] **2-2. イベントフラグ参照の未実装**
  - `EventChoiceData.sets_flag` でフラグを立てる機能は **実装済み**（`event_screen.gd:118`）
  - しかし `_check_requirement()` に `flag==` 系の条件パーサーが **存在しない**
  - → フラグを立てても後続イベントの出現条件として参照できない
  - 修正: `_check_requirement()` に `flag==<flag_name>` パターンを追加
- [ ] **2-3. イベント戦闘の固有敵指定**
  - `triggers_combat` 時の敵選出が「該当Actからランダム1体」のみ（`event_screen.gd:129-136`）
  - イベント固有の敵と戦わせる手段がない
  - → `EventChoiceData` に `combat_enemy_ids: Array[StringName]` フィールド追加を検討
- [ ] **2-4. イベント数の充実**
  - 23件は5区間のゲームとしては不足（区間あたり平均4.6件）
  - 特に travel カテゴリ（4件）とfaction（3件）が薄い
  - キャラ固有イベントの偏り確認（コンカラー・ヘドニスト向けが十分か）
- [ ] **2-5. クエスト数の拡充**
  - `resources/quests/` にあるクエストが `devilf_pack` の **1件のみ**
  - QuestManager の設計（hunt / poison / lapse / ボス修飾）は高機能だがコンテンツが不足
